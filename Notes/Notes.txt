1. What is Reconciliation?

In React, Reconciliation is:

The process of comparing the previous Virtual DOM with the new Virtual DOM and updating only the changed parts in the real DOM.

üß† Why React doesn‚Äôt update DOM directly?

Because:

Real DOM updates are slow

React creates a Virtual DOM (JavaScript object)

It compares changes in memory (fast)

Then updates only necessary nodes

Example:
root.render(<h1>Hello</h1>);

Later:

root.render(<h1>Hello Gafoor</h1>);

React compares:

Old:

<h1>Hello</h1>

New:

<h1>Hello Gafoor</h1>

It sees:

Same tag (h1)

Only text changed

So it updates only the text node, not the full element.

That comparison process = Reconciliation

3Ô∏è‚É£ What is Concurrent Rendering? (React 18 Feature)

In React 18, we have:

const root = ReactDOM.createRoot(container);

This enables Concurrent Rendering

What does that mean?

React can:

Pause rendering

Continue later

Prioritize important updates

Keep UI responsive

Example:

User typing in input (high priority)

Big list rendering (low priority)

React will prioritize typing first.
